📋 陳接手開發的技術清單與說明
為了讓 陳 能無縫接手並順暢聯調，請將以下技術重點與他同步：

1. ⚙️ 開發伺服器啟動指令
本地開發時，必須使用 Webpack {Webpack 編譯工具} 啟動 dev server {開發伺服器}，以避免 Next.js 的 Turbopack 動態路由解析 Bug（會導致動態 API 路由 404）：

bash
npm run dev -- --webpack
# 或
npx next dev --webpack
2. 🔐 Spotify 開發者後台配置
配合 Spotify 2025 年實施的安全新政策（禁止使用 localhost），陳需要在 Spotify Developer Dashboard {Spotify 開發者後台} 的應用程式設定中：

將 Redirect URIs {重導向網址清單} 設定為：
text
http://127.0.0.1:3000/api/spotify/callback
在本地開發瀏覽網站時，請一律使用 http://127.0.0.1:3000 作為瀏覽器網址，否則會因 Host 網域不一致導致 Cookie 遺失而無法成功綁定。
3. 🛡️ 自訂 Account Linking 流程防護
新實作的自訂綁定流程已由後端處理安全防範：

/api/spotify/auth：使用者點擊時會將 callbackUrl 與加密的 state {安全雜湊值} 寫入短效的 HttpOnly Cookie。防呆機制：如果使用者以 localhost 進入，頁面會自動呈現友善的 HTML 提示，引導使用者切換至 127.0.0.1，完全避免 Spotify 拋出錯誤。
/api/spotify/callback：回呼時會先校驗 state 以防範 CSRF {跨站請求偽造} 攻擊，隨後透過 getServerSession 獲取當前登入者 ID，並以 prisma.account.upsert 完成綁定，最後自動重導向回原頁面（如 onboarding 或 playlist/edit）。該流程確保了測試虛擬帳號也能順暢關聯真實的 Spotify 帳號資料。
