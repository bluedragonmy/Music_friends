# 專案任務清單 (Task List)：音樂數據與限時品味交流 APP

## Phase 1: 專案建置與基礎架構 (Infrastructure & Setup)
- [x] 初始化開發環境，建構前端 (Web/App) 與後端 API 專案骨架
- [x] 建立核心資料庫結構 (User, Playlist, Match, Chatroom, SyncLog, Tag 等)
- [x] 實作核心身分登入功能 (Google OAuth)
- [x] 實作第二階段帳號綁定 (Account Linking) 頁面與音樂平台 (Spotify) 連動授權
- [x] 實作 Middleware 保護層與 Spotify Token 自動更新機制

## Phase 2: 第一階段 - 配對與歌單管理模組 (Matching & Playlist)
- [x] 介接串流平台 API，開發讓使用者挑選 5 首歌的匯入機制 (API 與前端已完成)
- [x] 實作自訂歌曲「精華 30 秒區間」之擷取與儲存邏輯 (API 與前端已完成)
- [x] 建置每日配對演算法引擎 (隨機探索 / 相似曲風權重)，每日派發 5 位候選人
- [ ] 實作配對探索介面：支援隨時自由滑動「喜歡/跳過」按鈕，並實作單一專輯封面主色調渲染全域背景

## Phase 3: 第二階段 - 聊天室與限時互動模組 (Chat & Interaction)
- [/] 實作配對判斷邏輯：達成雙向喜歡時觸發建立配對房間 (API已完成)
- [/] 建構 WebSocket 即時連線伺服器，配合前端實作限時 48 小時聊天室與倒數計時器 (Socket 伺服器已建置，缺乏前端)
- [ ] 在聊天室介面實作查閱對方「歌單快照」與觸發「同步聆聽」喚醒原生 App 的功能
- [x] 開發 48 小時過期後之自動鎖定輸入權限功能 (已實作於 Socket)
- [x] 實作雙盲選擇表單結算 (保留關係 / 結束關係) 與相關阻擋/檢舉等安全保護機制 (已實作於 Socket)

## Phase 4: 第三階段 - 音樂日誌與數據管理模組 (Music Log & Data)
- [x] 設計並撰寫伺服器定時排程服務 (Cron Job)，背景定期提取使用者於串流平台的聆聽 Metadata
- [x] 建置數據分析邏輯，支援動態切換時間維度（日/週/月/季/年），彙整使用者的總聆聽時長與 Top 5 曲風排行
- [ ] 實作前端資料視覺化儀表板，產出專屬的「心智圖」與高質感 Glassmorphism 音樂日誌軌跡報表
- [ ] 開發「社群匯出分享圖卡」功能與一鍵切換至「交友歌單」的捷徑

## Phase 5: 第四階段 - 測試、優化與發佈 (Testing & Deployment)

- [ ] 針對跨平台 (iOS/Android/Web) 音訊預覽與同步播放進行設備相容性與極端點擊測試
- [ ] 執行 WebSocket 連線限流與 Redis 狀態快取的壓力極限測試
- [ ] 建立完善的環境整合與 CI/CD 自動化建置、部署流程
